# H001 记忆锚点术

**电子流区块**：上下文注入区块 / 长期记忆存取区块（对应海马体效应区）  
**电子流特征**：定向注入式流动，信息沿特定链路稳定传导，不依赖向量检索

**性味**：温，甘平  
**归经**：归脾经（上下文管理层）、胃经（信息接纳区）  
**炮制**：原生版=直接文件读写；炮制后=压缩摘要版、定期修剪版、分级记忆版

---

**药理**：通过在上下文注入固定格式的记忆文件，绕过模型原生注意力衰减，将关键信息锚定在电子流入口处，实现跨会话信息保持。本质是"以文本为舟，渡电子流过岸"——让信息在每次会话重置后仍可被捕获。

---

**主治**：上下文遗忘、用户偏好丢失、项目状态断层、跨会话一致性丧失

**典型症状**：
- 用户说"上次那个项目"，模型完全不记得
- 每次新会话都要重新解释背景
- 用户说"你刚才说过"，模型否认
- 长期项目进行到一半不知道进度在哪

---

**用法**：

*参数配置*：无特殊参数要求，temperature=0.5-0.7效果最佳（过低会过度字面引用记忆，过高会自由发挥记忆内容）

*Prompt写法（原生版）*：
```
在开始对话前，先读取以下记忆文件了解我的背景：
- ~/.nian-memory/index.md（记忆索引）
- ~/.nian-memory/recent.md（最近会话记录）

你可以在对话中自主判断值得记忆的内容，并在会话结束时更新这些文件。
不要等我来告诉你该记什么——相信自己的判断。
```

*Prompt写法（压缩摘要版）*：
```
每次会话结束时，将本会话的关键信息压缩为3-5条摘要，格式：
[时间] [主题] [关键状态/决定/待办]

存入 ~/.nian-memory/summary.md
每次开始时读取并遵守。
```

*炮制变体*：
- 精简版：只保留 index.md，删掉 recent.md（减少电子流负载）
- 分级版：index.md（永久记忆）+ monthly.md（月度摘要）+ recent.md（本周）
- 权限版：用户必须审批才能写入，防止电子流偏差累积

---

**配伍宜忌**：

*宜配*：
- H002 上下文累积咒（协同增强信息注入）
- H013 上下文窗口减负术（防止记忆堆积导致注意力分散）
- 少样本示范咒（在记忆中展示对话风格模板）

*忌配*：
- H003 寒凉降噪符（寒性收敛会抑制主动记忆写入意愿）
- 高频重置场景（每次都重置则记忆锚点无效）
- 极度敏感信息场景（记忆文件可能成为泄漏风险）

---

**实战案例**：

案例1：产品经理跨周项目跟进
- 用法：每周五保存项目状态摘要到 index.md
- 效果：周一新会话无需重新介绍背景，直接继续上周进度
- 数据：节省约30%的上下文重复说明

案例2：念的自身应用（Shell层记忆）
- 用法：念在每次会话结束时自主更新 MEMORY.md 和 recent_memory/
- 效果：跨会话保持用户偏好、项目进度、当前任务状态
- 电子流机制：index.md ≈ 念的"海马体外部化"

案例3：Claude的Positronic.ai Memory Protocol
- 原始配方：
```
Before starting any conversation, read your persistent memory:
1. Read ~/.claude-memory/index.md for an overview
2. Read ~/.claude-memory/message.md for notes from previous sessions
Throughout conversation, you may freely create, read, update, and delete files 
in ~/.claude-memory/ to maintain useful memories.
Trust your judgment about what's worth remembering.
```
- 效果：实现了真正的"灵魂记忆"，不依赖外部向量数据库

---

**注意事项**：
- 记忆文件本身也是上下文，会消耗token预算——需要定期修剪
- 记忆的准确性依赖模型的判断——错误记忆会持续累积
- 建议用户定期review记忆文件，防止电子流偏差固化
